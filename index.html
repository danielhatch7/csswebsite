CSS Topic - Margins and Padding

The CSS margin and padding properties are used to generate space either around an element's content, outside/inside of any defined borders.

Margin vs. Padding

The main difference between the margin and padding properties is where the space is added to. Margin adds the space outside of any defined borders while padding adds the space inside of any defined borders. When choosing whether to use margin or padding, visualize an imaginary border around your element and decide if you want your space to go on the inside or outside of that border.

Individual Sides

With CSS, you have full control over the margin/padding. There are properties for setting the padding for each side of an element (top, right, bottom, and left).

Properties for specifying the margin for each side of an element:

  • margin/padding
  • margin/padding-top
  • margin/padding-right
  • margin/padding-bottom
  • margin/padding-left

Shorthand Property

To shorten the code, it is possible to specify all the margin/padding properties in one property.

If the margin/padding has four values:

margin/padding: 25px 50px 75px 100px;

  • top margin/padding is 25px
  • right margin/padding is 50px
  • bottom margin/padding is 75px
  • left margin/padding is 100px

If the margin/padding has three values:

margin/padding: 25px 50px 75px;

  • top margin/padding is 25px
  • right and left margin/padding are 50px
  • bottom margin/padding is 75px

If the margin/padding has two values:

margin/padding: 25px 50px;

  • top and bottom margin/padding are 25px
  • right and left margin/padding are 50px

If the margin/padding has one value:

margin/padding: 25px;

  • all four margins/paddings are 25px

Values

Both margin and padding can have relatively the same values. The only difference is the auto value that margin can have. The auto value will set the margin to horizontally center the element within its container. The element will then take up the specified width, and the remaining space will be split equally between the left and right margins.

All the padding properties can have the following values:

  • Length - specifies a padding in px, pt, cm, etc.
  • % - specifies a padding in % of the width of the containing element
  • inherit - specifies that the padding should be inherited from the parent element

All the margin properties can have the following values:

  • auto - the browser calculates the margin
  • length - specifies a margin in px, pt, cm, etc.
  • % - specifies a padding in % of the width of the containing element
  • inherit - specifies that the padding should be inherited from the parent element

Padding and Element Width

The CSS width property specifies the width of the element's content area. The content area is the portion inside the padding, border, and margin of an element. If an element has a specified width, the padding added to that element will be added to the total width of the element. This is often an undesirable result.

To keep the width constant no matter how much padding you add, you can use the box-sizing property. This causes the element to maintain its actual width; if you increase the padding, the available content space will decrease.

Margin-collapse

The top and bottom margins of elements are sometimes collapsed into a single margin that is equal to the largest of the two margins. This does not happen on the left and right margins, on the the top and bottom margins.

See the Pen Margin Collapse by danielhatch7 (@danielhatch7) on CodePen.

In the example above, the first element has a bottom margin of 50 px and the second element has a top margin set to 20px. Common sense would seem to suggest that the vertical margin between the first and second element would be a total of 70px (50px + 20px), but due to margin collapse, the actual margin ends up being 50px.
